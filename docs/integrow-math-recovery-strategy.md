## 1. The Math Recovery Learning Framework (SEAL)

SEAL = **Stages of Early Arithmetical Learning**, developed by Robert J. Wright. It classifies how children progress through counting and arithmetic:

| SEAL Stage | Name | What the Student Does |
|------------|------|----------------------|
| Stage 1 | **Emergent** | Recognizes numerals, beginning to understand that numbers represent quantities. Cannot yet count items reliably. |
| Stage 2 | **Perceptual** | Counts items they can see/touch/hear. Always starts from 1 (can't count on). Needs physical objects present. |
| Stage 3 | **Figurative** | Counts concealed items but still starts from 1. Can visualize quantities but reverts to count-all. |
| Stage 4 | **Counting On** | Starts from a given number and counts forward/backward. No longer needs to start from 1. |
| Stage 5 | **Facile** | Uses grouping, partitioning, and mental strategies without counting by ones. Thinks in composites. |

### Mapping SEAL → VINCULUM Levels

| SEAL Stage | VINCULUM Level | Pattern Machine Equivalent |
|------------|---------------|---------------------------|
| Emergent + Perceptual | **Beginning** | Copies AB patterns with objects |
| Figurative | **Partial Understanding** | Extends patterns with occasional errors |
| Counting On | **Proficient** | Identifies repeating unit, extends independently |
| Facile | **Mastery** | Creates, describes, and explains pattern rules |

**Key insight:** VINCULUM's 4-tier system maps well to SEAL. The Beginning level collapses Emergent and Perceptual (both need concrete support), while Mastery aligns with Facile (mental flexibility, no scaffolding needed).

## 2. The 7 Mental Math Strategies

From Math Recovery / Integrow's research, these are the seven essential strategies students need for multi-digit computation. Each uses the **empty number line** as a recording tool.

### Strategy 1: Jump
**How it works:** Start at one number, jump by tens, then jump ones.
**Example:** 46 + 23 → Start at 46, jump +10 to 56, +10 to 66, +3 to 69.
**Strength:** Low error rate. Works for both addition and subtraction, regrouping or not.
**Number line visual:** Arcs showing each jump above the line.

### Strategy 2: Split
**How it works:** Decompose both numbers by place value, combine like places, recombine.
**Example:** 46 + 23 → 40+20=60, 6+3=9, 60+9=69.
**Strength:** Leverages place value understanding.
**Caution:** Can cause errors during regrouping (e.g., 47+28 → 60+15 requires extra step).

### Strategy 3: Split and Jump (Hybrid)
**How it works:** Split the tens, add tens, then jump the ones from the first number.
**Example:** 47 + 26 → 40+20=60, then 60+7=67, then +6 more → 67+3=70+3=73.
**Strength:** Combines the reliability of Jump with the efficiency of Split.

### Strategy 4: Jump to the Decuple
**How it works:** Jump to the nearest multiple of 10 first, then continue.
**Example:** 49 + 25 → 49+1=50, 50+20=70, 70+4=74.
**Strength:** Makes computation easier by hitting "friendly" numbers.
**Connection:** This is the "make a ten" strategy scaled up to larger numbers.

### Strategy 5: Over-Jump (Compensation)
**How it works:** Round one number up to a decuple, compute, then compensate.
**Example:** 47 + 29 → 47+30=77, 77-1=76.
**Strength:** Especially powerful for numbers near a decuple (29, 38, 51, etc.).

### Strategy 6: Compensation
**How it works:** Adjust one or both numbers to simplify, then compensate the answer.
**Example:** 47 + 26 → 50+26=76, 76-3=73.
**Distinction from Over-Jump:** Compensation can adjust either number; Over-Jump specifically rounds the second number up.

### Strategy 7: Transformation
**How it works:** Change both numbers to create an equivalent but easier problem.
**Example:** 47 + 26 → 50 + 23 = 73 (moved 3 from second to first).
**Key idea:** The total stays the same because you add to one what you take from the other.
**For subtraction:** 82 - 47 → 85 - 50 = 35 (add same amount to both).

## 3. Ready Set Math Modules (RSM)

Integrow's Ready Set Math is a digital resource with ~50-70 mini-lessons per module, formative assessments, and companion manipulative kits.

| Module | Topic | Grade Range | Key Manipulatives |
|--------|-------|-------------|-------------------|
| RSM01 | Early Numeracy: Words, Symbols, Quantities | Pre-K–1 | Counters, dice, numeral cards, dot plates |
| RSM02 | Add/Sub: Counting-Based Strategies | K–2 | Counters, ten-frames, number lines |
| RSM03 | Add/Sub to 10: Non-Count-by-One | K–2 | Ten-frames, arithmetic rack (rekenrek), dot cards |
| RSM04 | Add/Sub to 20: Non-Count-by-One | 1–3 | Arithmetic rack, twenty-frame, bead strings |
| RSM05 | Place Value & Base-10 System | 1–3 | Base-10 strips, bundling sticks, numeral rolls, 100-bead string |
| RSM06 | Add/Sub to 100: Mental Strategies | 2–4 | 100-bead string, base-10 strips, numeral rolls, empty number line cards |
| RSM07 | Mult/Div: Counting-Based Strategies | 2–4 | Counters, arrays, equal group cards |
| RSM08 | Mult/Div: Strategies & Basic Facts | 3–5 | 100-dot arrays, L-screens, N-Tile sets, spinners |

## 5. New App Proposals

### Priority 1: Critical Gaps (No current coverage)

#### App: `rekenrek` (Grades K-2)
**RSM Modules:** RSM03, RSM04
**Description:** Interactive arithmetic rack with 2 rows of 10 beads (5 red, 5 white per row). Students slide beads to build numbers, see 5/10 benchmarks, discover addition/subtraction strategies.
**Explore:** Free slide beads, see number update, discover 5-groups
**Practice:** "Show me 7" — student slides beads, multiple correct arrangements
**Play:** Flash a bead arrangement for 2 seconds → student types the number (subitizing)
**Standards:** K.OA.3, K.OA.4, 1.OA.6
**Leveling:**
- Beginning: Numbers 1-5 on one row
- Partial: Numbers 1-10 on one row
- Proficient: Numbers 1-20 across two rows
- Mastery: Addition/subtraction using both rows with strategy identification

#### App: `bead-string` (Grades 1-3)
**RSM Modules:** RSM05, RSM06
**Description:** 100-bead string in groups of 10 (alternating colors). A linear model bridging concrete counting to number-line thinking.
**Explore:** Drag to highlight beads, see count update, discover tens grouping
**Practice:** "Jump to 47" — student highlights correct beads
**Play:** "What is 38 + 25?" — student shows the jump on the bead string
**Standards:** 1.NBT.2, 2.NBT.1, 2.NBT.5
**Leveling:**
- Beginning: Count to find numbers 1-20
- Partial: Locate numbers 1-100 using tens groups
- Proficient: Show addition jumps on the string
- Mastery: Show subtraction and mixed operations

#### App: `mental-strategy-selector` (Grades 2-4)
**RSM Module:** RSM06
**Description:** Given a computation problem, student selects the best mental strategy and demonstrates it on an empty number line. Teaches all 7 strategies explicitly.
**Explore:** Watch animated demonstrations of each strategy
**Practice:** Given 47+29, choose a strategy, then draw jumps on an empty number line
**Play:** Speed round — which strategy is most efficient for this problem?
**Standards:** 2.NBT.5, 2.NBT.6, 3.NBT.2
**Leveling:**
- Beginning: Jump strategy only, no regrouping
- Partial: Jump + Split, some regrouping
- Proficient: All 7 strategies, student chooses
- Mastery: Efficiency analysis — "which strategy uses fewest steps?"

#### App: `empty-number-line` (Grades 1-4)
**RSM Module:** RSM04, RSM06
**Description:** An open-ended empty number line where students place points and draw arcs to record their mental computation. Teacher can see the strategy the student chose.
**Explore:** Free draw — place numbers, draw arcs, explore
**Practice:** "Solve 63 - 28 and show your jumps"
**Play:** Compare your strategy to a partner's — both correct, different paths
**Standards:** 1.OA.1, 2.NBT.5, 3.NBT.2

### Priority 2: Enhancements to Existing Manipulative Coverage

#### App: `ten-frame-builder` (Grades K-1)
**RSM Module:** RSM01, RSM02
**Description:** Interactive ten-frame(s) with draggable counters. Currently partially covered by `number-bonds` and `teen-number-factory` but not as a standalone manipulative tool.
**Key features:** Single frame (1-10), double frame (11-20), subitizing flash mode
**Standards:** K.CC.5, K.OA.3, K.OA.4

#### App: `array-explorer` (Grades 3-5)
**RSM Module:** RSM08
**Description:** Enhanced array tool with L-screen overlay. Student can cover parts of a 10×10 array to discover partial products, distributive property, and fact strategies.
**Key feature:** L-screen slides over a dot array to show, e.g., 7×8 = 5×8 + 2×8
**Standards:** 3.OA.5, 3.OA.7, 4.NBT.5

### Priority 3: Module-Inspired Extensions

#### App: `numeral-roll` (Grades K-2)
**RSM Module:** RSM01, RSM05
**Description:** Scrolling number tape (0-120+) where students locate, order, and find numbers. Supports: "find the number between," "what comes before/after," and hundred-chart patterns.

#### App: `equal-groups` (Grades 2-3)
**RSM Module:** RSM07
**Description:** Build multiplication understanding through equal groups (bags of apples, rows of desks) before moving to arrays. Bridges counting-based to multiplicative thinking.

#### App: `fact-strategy-lab` (Grade 3-5)
**RSM Module:** RSM08
**Description:** Multiplication fact strategies explicitly taught: doubles, double+1, near-squares, make-a-ten, halve-and-double. Each strategy demonstrated with array visuals.

## 6. Hub-Wide Leveling Rollout Plan

### What Exists Today
- **vinculum-adaptive-hook.js**: Rolling 8-problem adaptive engine (85% up / 40% down)
- **vinculum-dashboard.js**: Full analytics API with misconception logging
- **Pattern Machine**: Complete 3-layer leveled system (BENCHMARKS + LEVEL_CONFIG + Quick Quiz)
- **Hub metadata**: All tools have `ccss`, `diff`, `prereq`, `next` fields

### What's Needed: vinculum-levels.js Shared Module

A new shared module that any app can import to get the full leveled experience:

```
vinculum-levels.js
├── LevelSelector UI      → 3 buttons + locked Mastery
├── QuickQuiz engine      → 10-question adaptive placement
├── BenchmarkPanel        → Side panel with standard descriptors
├── LevelConfig reader    → Reads app-specific LEVEL_CONFIG
└── LocalStorage bridge   → Persists level per tool per student
```

### Per-App Requirements
Each app needs to provide:
1. **BENCHMARKS object** — 4-level descriptors for its standards (curriculum writing)
2. **LEVEL_CONFIG object** — What changes at each level (problem types, scaffolding, ranges)
3. **10 quiz questions** — Graded Q1-3 Beginning, Q4-6 Partial, Q7-9 Proficient, Q10 Mastery

### Rollout Priority (by Math Recovery alignment)
1. K-2 number sense apps (highest RSM alignment)
2. Operations apps with strategy teaching
3. Fraction and decimal apps
4. Geometry and measurement apps
5. Upper-grade algebra/stats apps
